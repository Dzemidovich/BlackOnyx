# Проверка автоматического обновления кэша

## Шаг 1: Откройте DevTools
1. Нажмите `F12` в браузере
2. Перейдите на вкладку **Network** (Сеть)
3. Обновите страницу (`F5`)

## Шаг 2: Проверьте версии файлов
Найдите в списке файлы CSS и JS и проверьте их URL:

### Должно быть (с версиями):
```
/css/components/cards.css?v=1746634800000
/js/app.js?v=1746634800000
/js/core/router.js?v=1746634800000
```

### НЕ должно быть (без версий):
```
/css/components/cards.css
/js/app.js
/js/core/router.js
```

## Шаг 3: Проверьте Console
1. Перейдите на вкладку **Console** в DevTools
2. Введите команду:
```javascript
window.APP_VERSION
```
3. Должно вывести большое число (timestamp)

## Если версии НЕ добавляются:

Это значит что скрипт cache-busting не работает. Возможные причины:

### Причина 1: Скрипт выполняется слишком рано
Скрипт пытается найти элементы до того как они загрузились.

### Причина 2: Браузер кэширует сам index.html
Даже если мы меняем index.html, браузер показывает старую версию.

## Решение: Проверьте сам index.html

1. В DevTools на вкладке Network найдите файл `index.html` или `localhost:8888`
2. Кликните на него
3. Перейдите на вкладку **Response**
4. Проверьте что там есть строки:
```html
<script>
    window.APP_VERSION = Date.now();
</script>
```

## Если этих строк НЕТ в Response:
Значит браузер показывает СТАРУЮ версию index.html из кэша!

### Решение:
1. Закройте браузер ПОЛНОСТЬЮ
2. Откройте снова
3. Или откройте в режиме инкогнито: `Ctrl + Shift + N`
4. Зайдите на `http://localhost:8888`

## Альтернативное решение: Очистка кэша через настройки

### Chrome/Edge:
1. `Ctrl + Shift + Delete`
2. Выберите "Кэшированные изображения и файлы"
3. Период: "Всё время"
4. Нажмите "Удалить данные"

### Firefox:
1. `Ctrl + Shift + Delete`
2. Выберите "Кэш"
3. Период: "Всё"
4. Нажмите "Удалить сейчас"
