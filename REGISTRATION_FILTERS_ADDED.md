# Фильтры для модерации регистраций

## Что добавлено

### 1. Фильтры статусов
- **На рассмотрении** (pending) - новые заявки, требующие модерации
- **Одобренные** (approved) - история одобренных регистраций
- **Отклоненные** (rejected) - история отклоненных заявок с причинами
- **Все** (all) - полный список всех регистраций

### 2. Backend изменения

**Новый endpoint:** `GET /api/users/registrations?status={status}`
- Возвращает регистрации по выбранному статусу
- Поддерживает фильтрацию: pending, approved, rejected, all
- Сортировка по дате модерации (новые сверху)

**Обновлен DTO:** `PendingRegistrationDto`
- Добавлено поле `RegistrationStatus`
- Добавлено поле `RejectionReason` (причина отклонения)
- Добавлено поле `ModeratedAt` (дата модерации)

### 3. Frontend изменения

**Фильтры:**
- Кнопки-чипы для переключения между статусами
- Активный фильтр подсвечивается зеленым
- Счетчик показывает количество записей по фильтру

**Карточки регистраций:**
- Бейдж статуса (На рассмотрении / Одобрено / Отклонено)
- Информация о дате модерации
- Причина отклонения (если есть)
- Кнопки действий только для pending статуса
- Для approved/rejected - информационное сообщение

**Стили:**
- Цветовая кодировка статусов (желтый/зеленый/красный)
- Адаптивная верстка фильтров
- Плавные переходы и анимации

## Как использовать

1. Откройте раздел "Модерация регистраций" в админ-панели
2. Используйте фильтры для просмотра:
   - Новых заявок (по умолчанию)
   - Истории одобренных пользователей
   - Истории отклоненных заявок с причинами
   - Всех регистраций сразу

## Технические детали

- Фильтр сохраняется в `registrationsModule.currentFilter`
- При переключении фильтра страница перезагружается с новыми данными
- Счетчик pending заявок в меню работает независимо от фильтра
- Одобренные/отклоненные записи остаются в базе для истории
