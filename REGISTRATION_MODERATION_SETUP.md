# Настройка системы модерации регистраций

## Что было добавлено

Система модерации позволяет администратору просматривать и одобрять/отклонять заявки на регистрацию новых пользователей.

### Функционал:
- ✅ Новые пользователи регистрируются со статусом "pending" (ожидает модерации)
- ✅ Администратор видит все заявки в разделе "Модерация регистраций"
- ✅ Отображаются все данные пользователя (кроме пароля)
- ✅ Для юридических лиц показываются полные реквизиты
- ✅ Администратор может одобрить или отклонить заявку с указанием причины
- ✅ Пользователь получает уведомление о результате модерации
- ✅ После одобрения пользователь может войти в систему

## Установка

### 1. Выполните SQL скрипт

Выполните файл `add_registration_status.sql` в вашей PostgreSQL базе данных:

```bash
psql -U postgres -d toolshopdb -f add_registration_status.sql
```

Или выполните напрямую в pgAdmin/DBeaver:

```sql
-- Добавляем поля для модерации
ALTER TABLE users 
ADD COLUMN IF NOT EXISTS registrationstatus VARCHAR(20) DEFAULT 'pending',
ADD COLUMN IF NOT EXISTS rejectionreason TEXT,
ADD COLUMN IF NOT EXISTS moderatedat TIMESTAMP,
ADD COLUMN IF NOT EXISTS moderatedby INT;

-- Создаём индекс
CREATE INDEX IF NOT EXISTS idx_users_registration_status ON users(registrationstatus);

-- Обновляем существующих пользователей (они уже одобрены)
UPDATE users 
SET registrationstatus = 'approved', 
    moderatedat = createdat 
WHERE registrationstatus IS NULL OR registrationstatus = 'pending';
```

### 2. Перезапустите приложение

После выполнения SQL скрипта обязательно перезапустите ASP.NET приложение.

### 3. Проверьте работу

1. Зарегистрируйте нового пользователя через форму регистрации
2. Войдите как администратор
3. Перейдите в раздел "Модерация регистраций"
4. Одобрите или отклоните заявку

## Использование

### Для пользователей:
1. Заполните форму регистрации
2. После отправки появится сообщение: "Ваша заявка отправлена на модерацию"
3. Дождитесь одобрения администратором
4. После одобрения войдите в систему

### Для администраторов:
1. Войдите в систему как Admin
2. Перейдите в "Модерация регистраций" в боковом меню
3. Просмотрите данные заявителя
4. Нажмите "Одобрить" или "Отклонить"
5. При отклонении укажите причину

## API Endpoints

### Получить список заявок на модерацию
```
GET /api/users/pending-registrations
Authorization: Bearer {admin_token}
```

### Одобрить регистрацию
```
POST /api/users/{id}/approve
Authorization: Bearer {admin_token}
```

### Отклонить регистрацию
```
POST /api/users/{id}/reject
Authorization: Bearer {admin_token}
Content-Type: application/json

{
  "reason": "Причина отклонения"
}
```

## Статусы регистрации

- **pending** - Ожидает модерации (по умолчанию для новых пользователей)
- **approved** - Одобрено (пользователь может войти)
- **rejected** - Отклонено (пользователь не может войти)

## Примечания

- Существующие пользователи автоматически получают статус "approved"
- Пользователи со статусом "pending" или "rejected" не могут войти в систему
- При одобрении/отклонении пользователь получает уведомление
- История модерации сохраняется (дата, модератор, причина отклонения)
