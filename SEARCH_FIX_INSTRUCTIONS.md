# Исправление Поиска Товаров

## Проблема
Поиск работал только по текущей странице (например, из 9000 товаров искал только по 20-30 на текущей странице).

## Что Было Исправлено

### 1. Регистронезависимый Поиск в Controllers/ProductsController.cs

**Проблема**: Поиск был чувствителен к регистру в PostgreSQL.

**Исправление**:
```csharp
// БЫЛО:
query = query.Where(p =>
    p.Name.Contains(searchTerm) ||
    p.Article.Contains(searchTerm) ||
    (p.Description != null && p.Description.Contains(searchTerm)));

// СТАЛО:
var lowerSearchTerm = searchTerm.ToLower();
query = query.Where(p =>
    p.Name.ToLower().Contains(lowerSearchTerm) ||
    p.Article.ToLower().Contains(lowerSearchTerm) ||
    (p.Description != null && p.Description.ToLower().Contains(lowerSearchTerm)));
```

Теперь поиск работает **регистронезависимо** - можно искать "дрель", "ДРЕЛЬ", "Дрель" - все будет работать.

### 2. Поиск По Всей Базе Данных

Фронтенд уже был настроен правильно:
- При активном поиске запрашивается `pageSize=1000` (вместо обычных 20)
- Это означает, что поиск идет по **всей базе данных**, а не только по текущей странице

## Как Работает Поиск Сейчас

### 1. Поиск с Главной Страницы (Dashboard)
- Введите запрос в строку поиска
- Появится выпадающий список с первыми 10 результатами (автодополнение)
- Нажмите Enter или кнопку "Найти"
- Откроется страница каталога с результатами поиска по **всей базе**

### 2. Поиск на Странице Каталога
- Откройте страницу "Товары"
- Введите запрос в поле "Поиск"
- Поиск автоматически применится ко **всем 9000 товарам**
- Результаты отобразятся на одной странице (до 1000 товаров)

### 3. Что Ищется
Поиск работает по трем полям:
- **Название товара** (Name)
- **Артикул** (Article)
- **Описание** (Description)

### 4. Приоритет Поиска
1. **Точное совпадение артикула** - если артикул точно совпадает, вернется только этот товар
2. **Точное совпадение названия** - если название точно совпадает, вернется только этот товар
3. **Частичное совпадение** - поиск по всем трем полям (название, артикул, описание)

## Производительность

### Оптимизация для Больших Баз Данных

Если у вас более 10,000 товаров, рекомендуется:

1. **Уменьшить pageSize при поиске** (в `wwwroot/js/modules/products.js`):
```javascript
// Текущее значение:
const pageSize = search ? 1000 : this.productsPerPage;

// Рекомендуется для больших баз:
const pageSize = search ? 100 : this.productsPerPage;
```

2. **Добавить индексы в базу данных**:
```sql
-- Индекс для поиска по названию
CREATE INDEX idx_products_name_lower ON products (LOWER(name));

-- Индекс для поиска по артикулу
CREATE INDEX idx_products_article_lower ON products (LOWER(article));

-- Индекс для поиска по описанию (GIN для полнотекстового поиска)
CREATE INDEX idx_products_description_gin ON products USING gin(to_tsvector('russian', description));
```

3. **Использовать полнотекстовый поиск PostgreSQL** (для очень больших баз):
```csharp
// В ProductsController.cs
query = query.Where(p => 
    EF.Functions.ToTsVector("russian", p.Name + " " + p.Article + " " + p.Description)
        .Matches(EF.Functions.ToTsQuery("russian", searchTerm)));
```

## Возможные Проблемы и Решения

### Проблема 1: Поиск Медленный
**Причина**: Нет индексов в базе данных  
**Решение**: Добавьте индексы (см. выше)

### Проблема 2: Не Находит Товары
**Причина**: Регистр не совпадает  
**Решение**: Уже исправлено - поиск теперь регистронезависимый

### Проблема 3: Находит Слишком Много Результатов
**Причина**: Поиск по описанию находит много совпадений  
**Решение**: Можно убрать поиск по описанию:
```csharp
// В ProductsController.cs
query = query.Where(p =>
    p.Name.ToLower().Contains(lowerSearchTerm) ||
    p.Article.ToLower().Contains(lowerSearchTerm));
    // Убрали: (p.Description != null && p.Description.ToLower().Contains(lowerSearchTerm))
```

### Проблема 4: Показывает Только 1000 Товаров
**Причина**: Ограничение pageSize=1000  
**Решение**: Увеличьте значение в `wwwroot/js/modules/products.js`:
```javascript
const pageSize = search ? 5000 : this.productsPerPage;
```

## Дополнительные Улучшения (Опционально)

### 1. Подсветка Результатов Поиска
Добавьте подсветку найденных слов в результатах:
```javascript
function highlightSearchTerm(text, searchTerm) {
    if (!searchTerm) return text;
    const regex = new RegExp(`(${searchTerm})`, 'gi');
    return text.replace(regex, '<mark>$1</mark>');
}
```

### 2. История Поиска

```javascript
// Отображение
const history = JSON.parse(localStorage.getItem('searchHistory') || '[]');

```

### 3. Поиск по Категориям
Добавьте фильтр категорий к поиску:
```javascript
// В loadProducts()
if (search && categoryId) {
    params.append('search', search);
    params.append('categoryId', categoryId);
}
```

### 4. Fuzzy Search (Нечеткий Поиск)
Для поиска с опечатками используйте библиотеку Fuse.js:
```javascript
// Установка
npm install fuse.js

// Использование
const fuse = new Fuse(products, {
    keys: ['name', 'article', 'description'],
    threshold: 0.3
});
const results = fuse.search(query);
```
