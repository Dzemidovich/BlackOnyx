# Bugfix Requirements Document

## Introduction

Данный документ описывает исправление двух критических багов в функционале управления заказами:

1. **Баг в корзине**: Кнопка "Оформить заказ" не работает - при нажатии ничего не происходит, модальное окно не открывается
2. **Баг в интерфейсе администратора**: Отсутствует кнопка "Детали заказа" для просмотра полной информации о клиенте

Эти баги критически влияют на основной бизнес-процесс (оформление заказов) и на возможность администратора получать полную информацию о клиентах для обработки заказов.

## Bug Analysis

### Current Behavior (Defect)

#### Баг 1: Неработающая кнопка оформления заказа

1.1 WHEN пользователь нажимает кнопку "Оформить заказ" в корзине THEN система не открывает модальное окно для ввода комментария

1.2 WHEN пользователь нажимает кнопку "Оформить заказ" THEN заказ не оформляется и пользователь остается на странице корзины

1.3 WHEN пользователь пытается оформить заказ THEN не происходит перенаправление на страницу "Заказы"

#### Баг 2: Отсутствие кнопки "Детали заказа" для администратора

1.4 WHEN администратор просматривает список заказов во вкладке "Заказы" THEN кнопка "Детали заказа" не отображается

1.5 WHEN администратор открывает детали заказа THEN регистрационные данные пользователя (физ. лица или юр. лица) не отображаются

1.6 WHEN администратор просматривает заказ от юридического лица THEN поля юридического лица (companyname, unp, legaladdress, actualaddress, bankname, bankcode, checkingaccount, directorname, contactphone, contactperson) не отображаются

### Expected Behavior (Correct)

#### Баг 1: Корректная работа кнопки оформления заказа

2.1 WHEN пользователь нажимает кнопку "Оформить заказ" в корзине THEN система SHALL открыть модальное окно с формой для ввода комментария к заказу

2.2 WHEN пользователь подтверждает оформление заказа в модальном окне THEN система SHALL создать заказ и отправить данные на сервер

2.3 WHEN заказ успешно оформлен THEN система SHALL перенаправить пользователя на страницу "Заказы"

2.4 WHEN заказ успешно оформлен THEN система SHALL показать уведомление об успешном оформлении с номером заказа

#### Баг 2: Отображение кнопки и данных для администратора

2.5 WHEN администратор (role === 'Admin' OR role === 'Manager') просматривает список заказов THEN система SHALL отображать кнопку "Детали заказа" около каждого заказа

2.6 WHEN администратор нажимает кнопку "Детали заказа" THEN система SHALL открыть модальное окно с полной информацией о заказе и клиенте

2.7 WHEN администратор просматривает детали заказа от физического лица (islegalentity === false) THEN система SHALL отображать: ФИО (userFullName), Email (userEmail)

2.8 WHEN администратор просматривает детали заказа от юридического лица (islegalentity === true) THEN система SHALL отображать: companyname, unp, contactperson, contactphone, legaladdress, actualaddress, directorname, bankname, bankcode, checkingaccount

2.9 WHEN администратор просматривает детали заказа THEN система SHALL NOT отображать пароль пользователя

2.10 WHEN обычный пользователь (role === 'Customer') просматривает свои заказы THEN система SHALL NOT отображать кнопку "Детали заказа" с регистрационными данными

### Unchanged Behavior (Regression Prevention)

3.1 WHEN пользователь добавляет товары в корзину THEN система SHALL CONTINUE TO корректно отображать список товаров с количеством и ценами

3.2 WHEN пользователь изменяет количество товара в корзине THEN система SHALL CONTINUE TO корректно пересчитывать итоговую сумму

3.3 WHEN пользователь удаляет товар из корзины THEN система SHALL CONTINUE TO корректно обновлять корзину

3.4 WHEN обычный пользователь просматривает свои заказы THEN система SHALL CONTINUE TO отображать кнопку "Детали" для просмотра состава заказа без регистрационных данных

3.5 WHEN администратор просматривает детали заказа THEN система SHALL CONTINUE TO отображать список товаров, статус заказа, дату создания и общую сумму

3.6 WHEN пользователь оформляет заказ с комментарием THEN система SHALL CONTINUE TO сохранять комментарий в базе данных

3.7 WHEN пользователь оформляет заказ без комментария THEN система SHALL CONTINUE TO создавать заказ с пустым комментарием

3.8 WHEN администратор фильтрует заказы по статусу THEN система SHALL CONTINUE TO корректно отображать отфильтрованные заказы
