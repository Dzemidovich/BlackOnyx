# Bugfix Requirements Document

## Introduction

Иерархические фильтры брендов с подгруппами не работают в каталоге товаров на http://localhost:8888. Система была полностью реализована (backend: HierarchicalFilterService.cs, FilterCacheService.cs, ProductsController.cs; frontend: catalog.js с методом toggleSubgroups(); стили: hierarchical-filters.css), компиляция успешна (0 ошибок), но при открытии каталога подгруппы брендов не отображаются или не раскрываются при взаимодействии пользователя.

Ожидаемое поведение: при клике на бренд (например, BOSCH) должны раскрываться подкатегории товаров внутри этого бренда (Дрели и шуруповерты, Болгарки (УШМ), Перфораторы, Остальное).

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN пользователь открывает каталог товаров на http://localhost:8888 и нажимает кнопку "Фильтры" THEN подгруппы брендов не отображаются в секции "Бренд"

1.2 WHEN пользователь видит бренд с кнопкой переключения (▼) и нажимает на неё THEN подгруппы не раскрываются или кнопка не реагирует на клик

1.3 WHEN API endpoint /api/products/filters возвращает данные с полем `subgroups` для брендов THEN frontend не рендерит эти подгруппы в DOM или рендерит их некорректно

1.4 WHEN метод `toggleSubgroups()` вызывается при клике на кнопку THEN подгруппы не становятся видимыми (display: none не меняется на display: block)

### Expected Behavior (Correct)

2.1 WHEN пользователь открывает каталог товаров и нажимает кнопку "Фильтры" THEN система SHALL отображать бренды с кнопкой переключения (▼) для брендов, у которых есть подгруппы

2.2 WHEN пользователь нажимает на кнопку переключения (▼) рядом с брендом THEN система SHALL раскрывать подгруппы с плавной анимацией и изменять иконку на (▲)

2.3 WHEN API endpoint /api/products/filters возвращает данные с полем `subgroups` THEN frontend SHALL корректно рендерить подгруппы в DOM с правильными data-атрибутами и обработчиками событий

2.4 WHEN метод `toggleSubgroups()` вызывается THEN система SHALL переключать видимость подгрупп (display: none ↔ display: block) и изменять иконку кнопки (fa-chevron-down ↔ fa-chevron-up)

2.5 WHEN пользователь выбирает чекбокс подгруппы THEN система SHALL фильтровать товары по выбранной подгруппе и её ключевым словам

### Unchanged Behavior (Regression Prevention)

3.1 WHEN пользователь фильтрует товары по другим атрибутам (не "Бренд") THEN система SHALL CONTINUE TO применять фильтры корректно без изменений

3.2 WHEN пользователь выбирает чекбокс бренда без раскрытия подгрупп THEN система SHALL CONTINUE TO фильтровать все товары этого бренда

3.3 WHEN API endpoint /api/products/filters возвращает атрибуты без подгрупп THEN система SHALL CONTINUE TO отображать их как обычные чекбоксы без кнопки переключения

3.4 WHEN пользователь применяет фильтры по цене, категории, наличию THEN система SHALL CONTINUE TO работать корректно независимо от состояния иерархических фильтров

3.5 WHEN пользователь сбрасывает фильтры кнопкой "Сбросить" THEN система SHALL CONTINUE TO очищать все фильтры включая выбранные подгруппы

3.6 WHEN кэш фильтров истекает (5 минут) THEN система SHALL CONTINUE TO перезагружать фильтры из базы данных с применением иерархической группировки
