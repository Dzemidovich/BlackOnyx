# Bugfix Requirements Document

## Introduction

Сайдбар в B2B платформе для продажи инструментов имеет несколько проблем с визуальным отображением:
1. Изображения товаров отображаются с белым фоном вместо прозрачного
2. Описание товара выдуманное (не соответствует реальным данным из БД)
3. Сайдбар слишком большой по размеру (нужно уменьшить по высоте и ширине)
4. Стилистика сайдбара не соответствует общему дизайну приложения

Эти проблемы влияют на пользовательский опыт и визуальную согласованность интерфейса.

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN изображение товара отображается в блоке `.sidebar-banner-image` THEN система показывает изображение с белым фоном вместо прозрачного

1.2 WHEN сайдбар рендерится на десктопной версии THEN система отображает сайдбар слишком большим по высоте и ширине (текущие размеры: ширина 64px в свернутом состоянии, 100px при наведении)

1.3 WHEN описание товара отображается в `.sidebar-banner-content` THEN система показывает выдуманное описание вместо реального названия товара из БД

1.4 WHEN сайдбар отображается THEN система использует стили, которые не соответствуют общей стилистике приложения (цвета, отступы, шрифты)

1.5 WHEN товары загружаются для отображения в сайдбаре THEN система может выбирать маленькие товары (не дрели, компрессоры и т.д.)

### Expected Behavior (Correct)

2.1 WHEN изображение товара отображается в блоке `.sidebar-banner-image` THEN система SHALL показывать изображение без фона (прозрачный фон или удаление белого фона через CSS)

2.2 WHEN сайдбар рендерится на десктопной версии THEN система SHALL отображать сайдбар уменьшенным в 2 раза по высоте блока `.sidebar-banner` и немного уже по ширине

2.3 WHEN описание товара отображается в `.sidebar-banner-content` THEN система SHALL показывать реальное название товара из поля `name` в таблице `Products`

2.4 WHEN сайдбар отображается THEN система SHALL использовать стили, соответствующие общей стилистике приложения (цвета из CSS переменных, правильные отступы, шрифты)

2.5 WHEN товары загружаются для отображения в сайдбаре THEN система SHALL выбирать случайный товар из категорий "Электроинструмент" или "Компрессоры" (крупные товары: дрели, компрессоры)

2.6 WHEN сайдбар отображается на мобильной версии THEN система SHALL применять похожее решение с уменьшенными размерами и правильным отображением изображений

### Unchanged Behavior (Regression Prevention)

3.1 WHEN пользователь наводит курсор на сайдбар THEN система SHALL CONTINUE TO расширять сайдбар и показывать текст навигационных элементов

3.2 WHEN пользователь кликает на элемент навигации в сайдбаре THEN система SHALL CONTINUE TO переходить на соответствующую страницу

3.3 WHEN сайдбар отображается для разных ролей (Admin, Manager, Customer) THEN система SHALL CONTINUE TO показывать соответствующие пункты меню для каждой роли

3.4 WHEN пользователь кликает на баннер товара в сайдбаре THEN система SHALL CONTINUE TO открывать детальную страницу товара

3.5 WHEN сайдбар отображается на мобильных устройствах (ширина < 768px) THEN система SHALL CONTINUE TO скрывать десктопный сайдбар и показывать bottom navigation

3.6 WHEN логотип и навигационные элементы отображаются в сайдбаре THEN система SHALL CONTINUE TO работать корректно без изменений в функциональности
