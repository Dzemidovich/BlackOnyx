# Bugfix Requirements Document

## Introduction

При подготовке приложения к развертыванию на production хостинге обнаружено отсутствие критически важных артефактов и функционала, необходимых для успешного деплоя и соответствия требованиям дипломного проекта.

Это критично для:
- Успешного развертывания на хостинге
- Создания и заполнения базы данных на production
- Демонстрации полного функционала на защите диплома
- Соответствия требованиям дипломного проекта (отчеты, экспорт данных)

## Bug Analysis

### Current Behavior (Defect)

1.1 WHEN разработчик пытается развернуть приложение на хостинге THEN отсутствует SQL скрипт с полной структурой БД для создания базы данных на хостинге

1.2 WHEN разработчик пытается заполнить БД тестовыми данными THEN отсутствует SQL скрипт с seed данными (товары, пользователи, заказы)

1.3 WHEN администратор хочет просмотреть отчеты по продажам THEN такой функционал отсутствует (требование диплома)

1.4 WHEN администратор хочет экспортировать данные THEN отсутствует функционал экспорта в CSV/Excel (требование диплома)

1.5 WHEN разработчик настраивает production конфигурацию THEN отсутствуют четкие инструкции по настройке переменных окружения и секретов

### Expected Behavior (Correct)

2.1 WHEN разработчик разворачивает приложение на хостинге THEN SHALL существовать SQL скрипт `Database_PRODUCTION.sql` с полной структурой всех таблиц

2.2 WHEN разработчик заполняет БД тестовыми данными THEN SHALL существовать SQL скрипт `Database_SEED_DATA.sql` с 20-30 товарами, 5-10 заказами, 3-5 пользователями

2.3 WHEN администратор открывает раздел "Отчеты" THEN SHALL отображаться отчет по продажам за период с возможностью выбора дат

2.4 WHEN администратор нажимает "Экспорт" в отчете THEN SHALL скачиваться CSV файл с данными отчета

2.5 WHEN разработчик настраивает production THEN SHALL существовать файл `DEPLOYMENT_INSTRUCTIONS.md` с пошаговой инструкцией

### Unchanged Behavior (Regression Prevention)

3.1 WHEN приложение работает локально в development режиме THEN SHALL CONTINUE TO использовать appsettings.Development.json и порт 8888

3.2 WHEN пользователь работает с каталогом товаров THEN SHALL CONTINUE TO работать поиск, фильтрация, сортировка

3.3 WHEN пользователь оформляет заказ THEN SHALL CONTINUE TO работать корзина и оформление заказа с комментариями

3.4 WHEN администратор управляет товарами THEN SHALL CONTINUE TO работать CRUD операции

3.5 WHEN администратор просматривает заказы THEN SHALL CONTINUE TO работать просмотр списка заказов и деталей заказа
