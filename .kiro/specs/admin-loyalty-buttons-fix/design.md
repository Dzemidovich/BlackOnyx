# Admin Loyalty Buttons Fix - Bugfix Design

## Overview

Кнопки управления скидками в админ-панели программы лояльности не работают из-за того, что объект `app` не доступен в глобальной области видимости `window`. HTML-код использует inline обработчики событий вида `onclick="app.openDiscountModal()"`, но JavaScript не может найти объект `app`, так как он определен как локальная переменная в `wwwroot/js/app.js`.

Исправление заключается в добавлении одной строки кода `window.app = app;` в конец файла `app.js` после определения объекта `app`, но до вызова `app.init()`. Это сделает объект доступным для inline обработчиков событий, сохраняя при этом всю существующую функциональность.

## Glossary

- **Bug_Condition (C)**: Условие, при котором проявляется баг - когда администратор нажимает кнопку с inline обработчиком `onclick="app.method()"`
- **Property (P)**: Желаемое поведение - метод должен успешно вызваться без ошибки "ReferenceError: app is not defined"
- **Preservation**: Существующая функциональность, которая должна остаться неизменной - все прямые обращения к `app` внутри JavaScript модулей
- **app**: Главный объект приложения, определенный в `wwwroot/js/app.js` как `var app = {...}`
- **window.app**: Глобальная ссылка на объект `app`, доступная из любого контекста, включая inline обработчики событий в HTML
- **openDiscountModal**: Метод в `wwwroot/js/pages/admin.js` (строка 5426), открывающий модальное окно для изменения скидки пользователя
- **openDiscountHistoryModal**: Метод в `wwwroot/js/pages/admin.js` (строка 5527), открывающий модальное окно с историей изменений скидок

## Bug Details

### Bug Condition

Баг проявляется, когда администратор нажимает на кнопки управления скидками в разделе "Пользователи" админ-панели. HTML-код использует inline обработчики событий, которые пытаются вызвать методы объекта `app`, но этот объект не доступен в глобальной области видимости.

**Formal Specification:**
```
FUNCTION isBugCondition(input)
  INPUT: input of type MouseEvent (click event)
  OUTPUT: boolean
  
  RETURN input.target.hasAttribute('onclick')
         AND input.target.getAttribute('onclick').includes('app.')
         AND typeof window.app === 'undefined'
END FUNCTION
```

### Examples

- **Пример 1**: Администратор нажимает кнопку "Скидка" (`onclick="app.openDiscountModal(123)"`) → Браузер выбрасывает `ReferenceError: app is not defined` → Модальное окно не открывается
- **Пример 2**: Администратор нажимает кнопку "История скидок" (`onclick="app.openDiscountHistoryModal(123)"`) → Браузер выбрасывает `ReferenceError: app is not defined` → Модальное окно не открывается
- **Пример 3**: Любая кнопка с `onclick="app.someMethod()"` → Ошибка JavaScript → Функциональность не работает
- **Edge case**: Если JavaScript код внутри модуля вызывает `app.openDiscountModal()` напрямую (не через inline обработчик) → Работает корректно, так как `app` доступен в локальной области видимости

## Expected Behavior

### Preservation Requirements

**Unchanged Behaviors:**
- Все существующие вызовы методов `app` внутри JavaScript модулей должны продолжать работать без изменений
- Навигация между страницами приложения должна работать корректно
- Все другие кнопки и элементы интерфейса должны функционировать как ожидается
- Расширение объекта `app` через `Object.assign(app, {...})` в других модулях должно работать корректно
- Инициализация приложения через `app.init()` должна выполняться без изменений

**Scope:**
Все взаимодействия с объектом `app`, которые НЕ используют inline обработчики событий в HTML, должны остаться полностью неизменными. Это включает:
- Прямые вызовы методов внутри JavaScript файлов (например, `this.openDiscountModal()` или `app.openDiscountModal()`)
- Event listeners, добавленные через `addEventListener`
- Любые другие программные вызовы методов объекта `app`

## Hypothesized Root Cause

На основе анализа кода, наиболее вероятная причина бага:

1. **Локальная область видимости**: Объект `app` определен как локальная переменная в `wwwroot/js/app.js` (строка 34: `var app = {...}`), что делает его недоступным в глобальной области видимости `window`

2. **Inline обработчики событий**: HTML-код использует inline обработчики вида `onclick="app.openDiscountModal()"`, которые выполняются в глобальном контексте и требуют, чтобы `app` был доступен как `window.app`

3. **Отсутствие глобального присваивания**: Нигде в коде не выполняется присваивание `window.app = app`, хотя в `wwwroot/js/modules/mobile-nav.js` (строка 13) есть строка `window.app = window.app || {};`, но она выполняется позже и создает пустой объект, если `window.app` еще не существует

4. **Порядок загрузки скриптов**: Файл `app.js` загружается первым, но не экспортирует объект `app` в глобальную область видимости, поэтому последующие inline обработчики не могут его найти

## Correctness Properties

Property 1: Bug Condition - Inline обработчики событий работают корректно

_For any_ HTML элемент с inline обработчиком события вида `onclick="app.method()"`, где метод существует в объекте `app`, исправленный код SHALL успешно вызвать этот метод без ошибки "ReferenceError: app is not defined", открывая соответствующее модальное окно или выполняя требуемое действие.

**Validates: Requirements 2.1, 2.2, 2.3, 2.4**

Property 2: Preservation - Существующие вызовы методов app

_For any_ вызов метода объекта `app` внутри JavaScript модулей (не через inline обработчики), исправленный код SHALL производить точно такой же результат, как и оригинальный код, сохраняя всю существующую функциональность навигации, расширения объекта через `Object.assign`, и инициализации приложения.

**Validates: Requirements 3.1, 3.2, 3.3, 3.4, 3.5**

## Fix Implementation

### Changes Required

Исходя из анализа первопричины, требуется минимальное изменение:

**File**: `wwwroot/js/app.js`

**Location**: После определения объекта `app` (строка ~759, после закрывающей скобки `};` объекта `app`), но ДО строки `document.addEventListener('DOMContentLoaded', function () {` (строка 762)

**Specific Changes**:
1. **Добавить глобальное присваивание**: Вставить строку `window.app = app;` после определения объекта `app`
   - Это сделает объект доступным в глобальной области видимости
   - Inline обработчики событий смогут найти объект `app` через `window.app`

2. **Проверить наличие всех необходимых методов**: Убедиться, что методы `openDiscountModal` и `openDiscountHistoryModal` определены в `wwwroot/js/pages/admin.js`
   - Метод `openDiscountModal` находится на строке 5426
   - Метод `openDiscountHistoryModal` находится на строке 5527
   - Оба метода добавляются к объекту `app` через `Object.assign(app, {...})`

3. **Порядок загрузки скриптов**: Убедиться, что `app.js` загружается перед `admin.js`
   - Это гарантирует, что объект `app` существует до того, как к нему добавляются методы через `Object.assign`

4. **Удалить конфликтующий код** (опционально): Рассмотреть удаление строки `window.app = window.app || {};` из `wwwroot/js/modules/mobile-nav.js` (строка 13)
   - Эта строка больше не нужна, так как `window.app` будет установлен в `app.js`
   - Однако, её можно оставить для обратной совместимости (она не вызовет проблем, так как `window.app` уже будет определен)

5. **Тестирование**: После внесения изменений проверить:
   - Кнопка "Скидка" открывает модальное окно без ошибок
   - Кнопка "История скидок" открывает модальное окно без ошибок
   - Все существующие функции приложения работают корректно

## Testing Strategy

### Validation Approach

Стратегия тестирования следует двухфазному подходу: сначала продемонстрировать баг на неисправленном коде, затем проверить, что исправление работает корректно и сохраняет существующее поведение.

### Exploratory Bug Condition Checking

**Goal**: Продемонстрировать баг ДО внесения исправления. Подтвердить или опровергнуть анализ первопричины. Если опровергнем, потребуется пересмотреть гипотезу.

**Test Plan**: Написать тесты, которые симулируют нажатие на кнопки с inline обработчиками событий. Запустить эти тесты на НЕИСПРАВЛЕННОМ коде, чтобы наблюдать ошибки и понять первопричину.

**Test Cases**:
1. **Test Discount Button Click**: Симулировать нажатие на кнопку "Скидка" с `onclick="app.openDiscountModal(123)"` (будет падать на неисправленном коде с ошибкой "ReferenceError: app is not defined")
2. **Test History Button Click**: Симулировать нажатие на кнопку "История скидок" с `onclick="app.openDiscountHistoryModal(123)"` (будет падать на неисправленном коде с ошибкой "ReferenceError: app is not defined")
3. **Test window.app Existence**: Проверить `typeof window.app` на неисправленном коде (должно быть "undefined" или "object" с пустым объектом)
4. **Test Direct Method Call**: Вызвать `app.openDiscountModal(123)` напрямую из JavaScript контекста (может работать на неисправленном коде, если `app` доступен в локальной области видимости)

**Expected Counterexamples**:
- Inline обработчики событий не могут найти объект `app` в глобальной области видимости
- Возможные причины: `window.app` не определен, объект `app` существует только в локальной области видимости

### Fix Checking

**Goal**: Проверить, что для всех входных данных, где выполняется условие бага, исправленная функция производит ожидаемое поведение.

**Pseudocode:**
```
FOR ALL input WHERE isBugCondition(input) DO
  result := handleInlineEventHandler_fixed(input)
  ASSERT expectedBehavior(result)
END FOR
```

**Testing Approach**: После добавления `window.app = app;` в `app.js`, проверить, что:
- `window.app` определен и является объектом
- `window.app.openDiscountModal` является функцией
- `window.app.openDiscountHistoryModal` является функцией
- Inline обработчики событий успешно вызывают эти методы

### Preservation Checking

**Goal**: Проверить, что для всех входных данных, где условие бага НЕ выполняется, исправленная функция производит тот же результат, что и оригинальная функция.

**Pseudocode:**
```
FOR ALL input WHERE NOT isBugCondition(input) DO
  ASSERT originalBehavior(input) = fixedBehavior(input)
END FOR
```

**Testing Approach**: Property-based testing рекомендуется для проверки сохранения поведения, потому что:
- Он автоматически генерирует множество тестовых случаев по всему входному домену
- Он ловит граничные случаи, которые могут быть пропущены в ручных unit тестах
- Он предоставляет сильные гарантии, что поведение не изменилось для всех не-багованных входных данных

**Test Plan**: Наблюдать поведение на НЕИСПРАВЛЕННОМ коде для прямых вызовов методов и других взаимодействий, затем написать property-based тесты, фиксирующие это поведение.

**Test Cases**:
1. **Direct Method Calls Preservation**: Наблюдать, что прямые вызовы `app.openDiscountModal()` внутри JavaScript модулей работают на неисправленном коде, затем написать тест для проверки, что это продолжает работать после исправления
2. **Navigation Preservation**: Наблюдать, что навигация между страницами работает на неисправленном коде, затем написать тест для проверки, что это продолжает работать после исправления
3. **Object.assign Preservation**: Наблюдать, что расширение объекта `app` через `Object.assign` работает на неисправленном коде, затем написать тест для проверки, что это продолжает работать после исправления
4. **Initialization Preservation**: Наблюдать, что `app.init()` выполняется корректно на неисправленном коде, затем написать тест для проверки, что это продолжает работать после исправления

### Unit Tests

- Тест проверки существования `window.app` после загрузки `app.js`
- Тест проверки наличия методов `openDiscountModal` и `openDiscountHistoryModal` в `window.app`
- Тест симуляции нажатия на кнопку с inline обработчиком `onclick="app.openDiscountModal()"`
- Тест симуляции нажатия на кнопку с inline обработчиком `onclick="app.openDiscountHistoryModal()"`
- Тест проверки, что прямые вызовы методов внутри JavaScript модулей продолжают работать

### Property-Based Tests

- Генерировать случайные ID пользователей и проверять, что `app.openDiscountModal(userId)` вызывается без ошибок
- Генерировать случайные ID пользователей и проверять, что `app.openDiscountHistoryModal(userId)` вызывается без ошибок
- Генерировать случайные состояния приложения и проверять, что навигация работает корректно
- Тестировать, что все не-inline вызовы методов продолжают работать во множестве сценариев

### Integration Tests

- Тест полного потока: загрузка админ-панели → переход в раздел "Пользователи" → нажатие на кнопку "Скидка" → открытие модального окна → изменение скидки → сохранение
- Тест полного потока: загрузка админ-панели → переход в раздел "Пользователи" → нажатие на кнопку "История скидок" → открытие модального окна → просмотр истории
- Тест проверки, что другие inline обработчики событий в приложении продолжают работать корректно
- Тест проверки, что визуальная обратная связь (модальные окна, уведомления) отображается корректно
