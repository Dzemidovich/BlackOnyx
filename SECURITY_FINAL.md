```

### Регистрация:
```
✅ Успешная регистрация - записывается email, IP
❌ Неудачная регистрация - записывается email, IP, причина

### Несанкционированный доступ:
```
❌ Попытка доступа к чужому заказу - записывается user ID, IP, endpoint

```

## 🔍 Как проверить логи:

### Просмотр логов:
```bash
# Windows
type logs\security.log

# Linux/Mac
cat logs/security.log

# Последние 50 строк
tail -n 50 logs/security.log
```

### Поиск подозрительной активности:
```bash
# Неудачные попытки входа
grep "LOGIN_ATTEMPT.*Success: false" logs/security.log

# Несанкционированный доступ
grep "UNAUTHORIZED_ACCESS" logs/security.log

# Активность с конкретного IP
grep "192.168.1.100" logs/security.log
```

## 🚨 Признаки атаки:

### 1. Брутфорс (перебор паролей)
```
Много неудачных попыток входа с одного IP за короткое время
```

### 2. IDOR атака (доступ к чужим данным)
```
Много UNAUTHORIZED_ACCESS от одного пользователя
```

### 3. Сканирование (поиск уязвимостей)
```
Запросы к несуществующим endpoint'ам
Необычные User-Agent
```

### 4. Credential Stuffing
```
Попытки входа с разных IP с одинаковыми паролями
```

## 🛡️ Что делать при обнаружении атаки:

### 1. Заблокировать IP
Добавьте в `Program.cs`:
```csharp
var blockedIps = new[] { "192.168.1.100", "10.0.0.50" };
app.Use(async (context, next) =>
{
    var ip = context.Connection.RemoteIpAddress?.ToString();
    if (blockedIps.Contains(ip))
    {
        context.Response.StatusCode = 403;
        return;
    }
    await next();
});
```

### 2. Временно отключить регистрацию
Закомментируйте метод `Register` в `AuthController.cs`

### 3. Усилить Rate Limiting
В `Program.cs` уменьшите лимиты:
```csharp
options.PermitLimit = 3; // было 5
options.Window = TimeSpan.FromMinutes(30); // было 15
```

### 4. Включить Captcha
Добавьте Google reCAPTCHA на формы входа/регистрации
