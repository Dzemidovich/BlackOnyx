# ДИАГНОСТИКА ПРОБЛЕМЫ С CSS

## Шаг 1: Откройте DevTools
1. Нажмите `F12`
2. Перейдите на вкладку **Network**
3. Поставьте галочку **Disable cache**
4. Перезагрузите страницу (`F5`)

## Шаг 2: Проверьте загрузку CSS
Найдите в списке файл `orders.css` и проверьте:
- ✅ Загружается ли файл? (статус 200 или 304)
- ✅ Какая версия загружается? (должно быть `?v=20241202001`)
- ✅ Есть ли ошибки 404?

## Шаг 3: Проверьте Console
Перейдите на вкладку **Console** и проверьте:
- ❌ Есть ли ошибки CSS?
- ❌ Есть ли ошибки загрузки файлов?

## Шаг 4: Проверьте Elements
1. Перейдите на вкладку **Elements**
2. Найдите элемент с классом `order-item-preview`
3. Посмотрите в правой панели **Styles**
4. Проверьте, какие стили применяются

## Что нужно сообщить:
1. Загружается ли `orders.css?v=20241202001`?
2. Какой статус у файла (200, 304, 404)?
3. Есть ли ошибки в Console?
4. Какие стили видны для `.order-item-preview` в Elements?

## Возможные проблемы:

### Проблема 1: Файл не загружается (404)
**Причина**: Сервер не отдает статические файлы
**Решение**: Перезапустить сервер

### Проблема 2: Старая версия (не v=20241202001)
**Причина**: Кэш браузера
**Решение**: Disable cache в DevTools + перезагрузка

### Проблема 3: Файл загружается, но стили не применяются
**Причина**: Синтаксическая ошибка в CSS или конфликт стилей
**Решение**: Проверить Console на ошибки

### Проблема 4: CSS переменные не работают
**Причина**: variables.css не загружается
**Решение**: Проверить загрузку variables.css
